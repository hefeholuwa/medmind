## Tracks

### Track 1: MemoryAgent
Build an Agent with **persistent memory** that autonomously accumulates experience, remembers user preferences, and makes increasingly accurate decisions across multi-turn, cross-session interactions.

**Focus on:**
- Efficient memory storage and retrieval
- Timely forgetting of outdated information
- Recalling critical memories within limited context windows

### Track 2: AI Showrunner
Leverage **video generation** capabilities (Wan / HappyHorse) to build an Agent that autonomously handles the entire short drama creation pipeline — scriptwriting → storyboarding → video generation → editing.

**Focus on:**
- Narrative ability and multimodal orchestration
- Output quality under limited token budget
- This track has the **highest token allowance**

### Track 3: Agent Society
Design a **multi-agent collaboration system** where multiple Agents with distinct capabilities work together through task division, dialogue, and negotiation.

**Focus on:**
- Task decomposition and role assignment
- Disagreement resolution and execution conflict handling
- Measurable efficiency gain over single-agent baselines

### Track 4: Autopilot Agent
Build an Agent that **automates real-world business workflows** end-to-end. Open-ended scenarios.

**Focus on:**
- Handling ambiguous inputs
- Invoking external tools
- Human-in-the-loop checkpoints at critical decision points
- **Production-readiness over toy demos**

### Track 5: EdgeAgent
Build Qwen-powered **physical devices** — robots, IoT agents, smart hardware — that perceive via edge sensors, reason via cloud APIs, and act locally.

**Focus on:**
- Robust edge-cloud orchestration under bandwidth/latency constraints
- Privacy-aware data handling
- Graceful degradation in offline/weak-network scenarios

## Submission Requirements

### Required:
1. **Code Repository URL** — public, open source (with detectable license file at top of repo)
2. **Proof of Alibaba Cloud Deployment** — link to a code file in repo demonstrating use of Alibaba Cloud services/APIs
3. **Architecture Diagram** — clear visual of how Qwen Cloud connects to backend, database, and frontend
4. **Demo Video** (≤ 3 minutes):
   - Shows the project functioning on its target platform
   - Uploaded to YouTube, Vimeo, or Youku (public)
   - No unauthorized third-party trademarks or copyrighted music
5. **Text Description** — features and functionality explanation
6. **Track Selection** — identify which track

### Optional:
7. **Blog/Social Post URL** — for Blog Post Prize eligibility (must show journey building with QwenCloud)

## Judging

### Stage 1: Pass/Fail
- Does the project fit the theme?
- Does it use the required APIs/SDKs?

### Stage 2: Weighted Criteria

| Criteria | Weight | What Judges Look For |
|---|---|---|
| **Innovation & AI Creativity** | 30% | Sophisticated use of Qwen Cloud APIs (custom skills, MCP integrations); algorithmic/engineering innovation; novel solutions, custom components, performance optimization |
| **Technical Depth & Engineering** | 30% | Architecture quality (modularity, scalability, error handling); clean code, non-trivial logic; advanced patterns, thoughtful tech stack adoption |
| **Problem Value & Impact** | 25% | Real-world relevance (solves authentic pain point); scalability potential (productization or open-source adoption) |
| **Presentation & Documentation** | 15% | Technical demo clarity (key logic visualized); clear architecture docs |

### Tie-breaking:
- Highest score in first criterion → second → third → etc.
- If still tied, judges vote
